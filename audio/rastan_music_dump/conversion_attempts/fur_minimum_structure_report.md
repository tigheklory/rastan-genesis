# Minimum Furnace `.fur` Structure Report

Evidence source: local `vgm2fur` serializer implementation and generated `.fur` artifacts.

## Required Structural Elements
1. Zlib-compressed file payload.
2. Decompressed header starts with `-Furnace module-`.
3. Version fields present (`TARGET_FURNACE_VERSION=228` in local serializer).
4. `INFO` chunk present with at least:
   - ticks per second
   - pattern length
   - order count
   - instrument count
   - pattern count
   - system id (Genesis-family in this serializer)
5. Asset directory chunks (`ADIR`) present.
6. Instrument chunks (`INS2`) present for referenced instruments.
7. Pattern chunks (`PATN`) present with event stream ending in `0xFF` end-of-pattern marker.
8. Channel order matrix and effect-column metadata present in INFO payload.

## Note/Event Encoding Findings
- Per-row event entries are encoded using mask+payload records generated by `Entry(...)`.
- OFF note code is `180` (`furnace.notes.Off`).
- Notes are serialized as one-byte note values in the Furnace note domain.

## Minimum Viable Generation Contract Used
- Non-empty instrument table with placeholder FM+PSG instruments.
- Non-empty pattern matrix with channel-assigned row events.
- Valid order/pattern linkage via Module serializer.
- Zlib compressed output from serializer (`Module.build()` default).

## Practical Validation Rules Applied
- File exists and size > 0.
- Zlib decompression succeeds.
- Decompressed stream contains `-Furnace module-` signature.
- `PATN` and `INFO` tags are present in decompressed stream.
- Generated outputs differ across tracks where source content differs.
